### How the Agent Works (Gstack Sprint Workflow)

Every task the client submits goes through the gstack sprint:

```
CLIENT SUBMITS: "Build a patient re-engagement campaign"

    ↓ /office-hours (YC framing)
    → Reframes: "You need a VIP patient win-back program"

    ↓ /plan-ceo-review (CEO finds 10x version)
    → "Target Elite+VIP patients with personalized video"

    ↓ /plan-eng-review (Architecture locked)
    → Plan: segment patients → generate video → send WhatsApp

    ↓ I BUILD (incremental implementation)
    → Thin slice: patient list first → test → next slice

    ↓ /review (Staff Engineer finds bugs)
    → Checks for N+1 queries, race conditions, bad error handling

    ↓ /qa (QA Lead tests in real browser)
    → Logs in as patient, tests full flow, finds bugs, fixes them

    ↓ /ship (Release Engineer)
    → Merges to main, deploys, verifies health

    ↓ /retro (Weekly retrospective)
    → "This campaign: 124 patients re-engaged, R48K revenue attributed"

    ↓ LLM-WIKI (Persistent memory)
    → Saved forever. Next campaign builds on this one.
```
